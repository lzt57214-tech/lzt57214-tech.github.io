Project 1 2025

Multi-Stock Price Prediction with LSTM Neural Network

Problem

Stock price forecasting is a core challenge in quantitative finance. The team aimed to build a single unified deep learning model capable of predicting next-day closing prices across hundreds of S&P 500 companies simultaneously — replacing company-specific models with one scalable, generalised system.

Data

Historical daily trading records for 505 S&P 500 companies (497 retained after quality filtering); 600,000+ total samples. Closing prices range from $1–$1,500 across all major sectors, introducing significant cross-company price-scale variance.

Approach

LSTM time-series model on standardised, company-encoded sliding-window sequences. Pipeline: EDA → StandardScaler normalisation → sequence construction → LSTM training with early stopping → inverse-transform evaluation on original price scale.

My Contribution

Contributed to EDA (price and sample distribution analysis across sectors), model evaluation framework design, and written analysis of error patterns across low-, medium-, and high-price stock segments.

Outcome / Impact

Model achieved R² = 0.9968, MAPE = 10.77%, RMSE = 7.26 on the test set — demonstrating strong trend-capture capability across diverse price ranges with a single unified architecture covering 505 companies.

Project 2 2025

Financial News Sentiment Classification via Fine-Tuned BERT

Problem

Manual interpretation of financial headlines is slow, inconsistent, and unscalable. The team built an automated three-class sentiment classifier (positive / neutral / negative) for financial news, with a live RSS inference extension enabling real-time market sentiment monitoring.

Data

Financial PhraseBank (Kaggle): 4,846 English financial sentences annotated by 16 finance-domain experts. Class distribution: 59.4% neutral, 28.1% positive, 12.5% negative. Stratified 80/10/10 train-val-test split with fixed seed for reproducibility.

Approach

Full fine-tuning of bert-tiny (Hugging Face) using AdamW optimiser with linear warmup scheduling, gradient clipping (max norm 1.0), and macro-F1-based early stopping. Live RSS pipeline connected to trained classifier for real-time inference on new headlines.

My Contribution

Led the writing of the full project report — covering system design, data pipeline documentation, model architecture, training methodology, quantitative evaluation, and error analysis of neutral-boundary misclassifications.

Outcome / Impact

Achieved test accuracy 75.88% and macro-F1 0.7322; neutral F1 = 0.8183, negative F1 = 0.7287. Early stopping at epoch 28 confirmed effective regularisation. Live RSS pipeline demonstrated real-world applicability for market sentiment monitoring.

Project 3 2025

CPU-Only Real-Time Mask Detection System (YOLOv8)

Problem

Existing mask detection systems require GPU hardware, limiting deployment in cost-constrained public venues. The team developed a lightweight, CPU-only detection system deployable on ordinary PCs — removing the GPU bottleneck for hospitals, shopping malls, and transport hubs.

Data

Custom mask dataset: 1,591 images (1,257 train / 334 val), 3,326 annotated instances in YOLO standard format. Significant class imbalance (unmask:masked = 3.14:1) was identified through statistical analysis and factored into training design.

Approach

Transfer learning fine-tuning of YOLOv8n (Ultralytics) on custom dataset, fully CPU-only. Three-tier modular architecture (Data → Algorithm → Application). Deliverables: CLI, offline image/video inference, and PyQt5 GUI with QThread multi-threading to prevent UI freezing.

My Contribution

Contributed to system architecture design, model training and validation pipeline, dataset statistical analysis (identified 3.14:1 class imbalance), performance benchmarking, and project documentation.

Outcome / Impact

Fine-tuned YOLOv8n achieved mAP@0.5 = 82.88%, mean precision = 88.65%, CPU inference speed = 37.13 FPS (26.94 ms/image) — meeting real-time requirements for deployment in public venues with zero GPU dependency.

Project 4 2026

Tiny-Helper: Intent Recognition & Tool Calling in a 0.1B Small Language Model

Problem

Mainstream tool-calling frameworks (function calling, ReAct) require models with tens of billions of parameters, making them impractical for edge devices and resource-constrained environments. The team explored whether a 0.1B small language model could autonomously recognise user intent and trigger external tool calls — validating the performance boundary of small-scale LLMs.

Data

Incremental pre-training: 7 million samples (Belle Chinese dataset + MiniMind original data, expanded from 1.4M). LoRA fine-tuning: 4,000 instruction samples (2,000 chat + 2,000 TTS-call intents) generated via GPT-4 with structured templates.

Approach

Built on MiniMind (0.143B causal LM). Pipeline: incremental pre-training on A100 (~10 hrs) → LoRA fine-tuning (rank=8, alpha=16) on RTX 4060 Laptop (~4.8 mins, 3 epochs). Model outputs structured JSON for TTS calls or plain text for chat, parsed by a lightweight inference layer.

My Contribution

Responsible for LoRA fine-tuning and inference — configured LoRA adapters across all linear modules (q/k/v/o, gate/up/down, lm_head), ran fine-tuning experiments, and implemented the inference pipeline for intent parsing and TTS invocation.

Outcome / Impact

Chat quality average score 0.778 (85% pass rate); intent recognition accuracy 70% with 100% recall for chat intent. Identified data diversity as the key bottleneck (TTS recall 40%), providing empirical insights for scaling small-model tool calling to more complex tasks.
